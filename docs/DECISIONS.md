## Decisions in chronological order (newest first)

- 2026-05-28 — Phase 3 shipped end-to-end + Phase 2.6 audio re-enabled; roadmap restructured.
    - **Phase 2.6 re-enable:** migrated `src/lib/audio/playback.ts` from `expo-av` → `expo-audio` (the SDK 54+ replacement), and pre-download the song via `expo-file-system` (legacy API) to a `file://` URI in the cache directory before playback. Earlier deferral (DECISIONS 2026-05-24) traced the failure to streaming a Supabase Storage signed URL (the 302 redirect + ambiguous `content-type` was the suspected cause). Playing from a local file sidesteps both. `HYPE_SONG_ENABLED` flipped back to `true`. Code-only fix — founder must verify on a real iPhone.
    - **Phase 3.1 cost-of-eating:** new `food_prices` table (community model) with RLS that lets any signed-in user read all rows but only insert/update/delete their own submissions. `verified_count` column kept on the schema for future server-side verification (Phase 5 / community feature) — v1 just uses `count(*)` per (food, region) as the suggestion strength. Prices normalised to "per 100 g" in storage so different package sizes compare cleanly; the log screen takes a total price (the natural way users think about it) and converts. Sharing-back logic only fires when there's no existing median or the user's per-100g price differs >30% from the median (cuts noise from duplicate identical entries). `profiles.region` + `currency` default from the device locale via `src/lib/locale/region.ts` (no new package — Intl + `NativeModules.SettingsManager`). `food_logs.price_at_log` + `currency_at_log` snapshot per master prompt § 5.
    - **Phase 3.2 freemium gating renumbered (was 3.3) and scaffolded:** `subscriptions` table with `tier`, `status`, `source`, `current_period_end`. RLS in v1: users can upsert their own row but ONLY when `source = 'stub'`. Phase 4 RevenueCat work will drop both stub policies and route writes through a SECURITY DEFINER function called by the webhook — `source = 'revenuecat'` rows will be untouchable by client. Free-tier limits centralised in `FREE_LIMITS` (`src/features/billing/types.ts`): hype songs capped at 3 distinct exercises, plan suggestions per slot 2 vs premium 5, advanced PPL split is Premium-only. The cost month view auto-opens the paywall on entry for non-premium and dismisses back to the previous screen. The stub `purchasePremium()` flips the row directly; Phase 4 deletes that and wires RevenueCat.
    - **Roadmap restructure** (master prompt § 8, `docs/ROADMAP.md`): Phase 3 = cost + freemium gating (only the in-app, code-only parts); new Phase 4 = "Credentialed launch" with Google/Apple OAuth completion (was 0.3 deferral), Spotify SDK, Apple MusicKit, and RevenueCat live billing — every Phase 4 task is blocked on a credential or account only the founder can provide; old Phase 4 "Scale & moat" renumbered to Phase 5. The renumber preserves the principle that streaming-music (caveat-laden, founder-credential-blocked) is decoupled from the rest of Phase 3, which is fully code-complete.
